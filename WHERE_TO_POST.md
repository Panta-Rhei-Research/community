# Where to Post What

Use this guide before opening a Discussion, Issue, or Pull Request.

## I have a general question

Use GitHub Organization Discussions:

- Start Here / Orientation
- Ask About the Framework

## I want to challenge a claim

Use GitHub Organization Discussions first.

Choose the closest category:

- Research Agenda
- Corpus & Construction Spine
- Results & World Readouts
- Predictions & Falsification
- TauLib / Formalization

A good challenge names:

- the page or artifact;
- the exact claim;
- what seems wrong or unsupported;
- what would count as failure;
- relevant evidence, citation, or counterexample.

## I found a concrete bug

Use Issues in the relevant repo.

Examples:

- broken link;
- wrong metadata;
- typo batch;
- bad redirect;
- raw template rendering;
- incorrect source mapping;
- build failure.

## I want to propose a correction

If it is conceptual or not yet concrete, start in Discussions.

If it is a concrete edit, open a Pull Request in the relevant repo.

## I want to review a bounded area

Use Discussions -> Review Offers / Domain Expertise.

Say:

- area of expertise;
- artifact or page to review;
- whether review can be public;
- what kind of review you can offer;
- any conflict or affiliation to disclose.

## I want to contribute code or documentation

Use:

- `site` for website/source changes;
- `taulib` for Lean/formalization;
- `publications` for publication artifacts;
- `research` for scripts, notebooks, or import reports;
- `community` for guidelines/templates/routing.

## I want private or institutional contact

Use email.

Email is appropriate for:

- institutional review;
- collaboration proposals;
- media;
- funding/support;
- sensitive critique;
- non-public context.
