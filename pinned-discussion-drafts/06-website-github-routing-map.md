# Website / GitHub Routing Map

The website and GitHub have different jobs.

The website is the public research observatory:

- Discover;
- Program;
- Corpus;
- Results;
- Verify;
- Publications;
- Impact;
- Engage.

GitHub is the public engagement and contribution substrate:

- Organization Discussions for public questions, critique, review offers, and routing;
- Issues for concrete defects;
- Pull Requests for concrete changes;
- `site` for website source;
- `publications` for publication artifacts;
- `taulib` for Lean formalization;
- `research` for scripts, notebooks, reports, and exploratory material;
- `community` for onboarding, routing, and participation guidance.
